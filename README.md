# DeepSeaToDo V2 – Abyss Build

![DeepSeaToDo Screenshot](DeepSeaTodo.png)

深海のように静かに、目の前のことを積む。  
学習用のTodoアプリから一歩進めて、**実際に使いながら改善したプロトタイプ**です。

## 概要

DeepSeaToDo V2 – Abyss Build は、JavaScript学習の中で制作したTodoアプリです。  
Todoの追加・削除・編集といった基本機能だけでなく、カテゴリ管理、期限表示、完了状態の視覚表現などを実装し、**使いやすさと世界観の両立**を意識して改善を重ねました。

「ただ作る」だけではなく、実際に触って違和感を見つけ、修正していく流れを通して育てた作品です。

## 制作目的

このアプリは、以下の力を伸ばすことを目的に制作しました。

- stateを中心にUIを更新する考え方の理解
- DOM操作とイベント処理の実践

## 工夫したポイント

### 1. stateを中心にUIを更新する構造
データの状態を `state` にまとめ、`setState()` を通して更新し、`render()` でUIに反映する形にしました。  
「UIはstateの投影」という考え方を意識して組んでいます。

### 2. 期限表示のロジック分離
期限に関する表示は `getDeadlineInfo()` に切り出し、render内の責務を軽くしました。  
これにより、見通しを保ちながら「明日」「あと◯日」「期限切れ」などの表示を実装しています。

### 3. 並び順の整理
`getSortedTodos()` を分けて、以下の優先順位で表示されるようにしました。

1. 未完了かつ期限あり
2. 未完了かつ期限なし
3. 完了タスク

使う側が自然に優先順位を判断しやすい並びを意識しています。

### 4. 世界観を意識したUI改善
深海をイメージしたカラーで統一し、完了タスクは少し暗く・静かに沈むような表現にしました。  
警告色も強すぎる赤ではなく、全体の雰囲気を壊しすぎない見せ方を意識しています。

### 5. 実際に使って違和感を修正
「文字が見えにくい」「期限情報をもっと分かりやすくしたい」「フィルターの現在位置が分かりづらい」など、  
使ってみて気づいた違和感を少しずつ修正しました。
